# Retros

The reasoning trail for agent-behavior learnings, written by the
[`create-retro`](../../../.agents/skills/create-retro/SKILL.md) skill
(`/retro`).

Each session that earned a retro — one with real back-and-forth, or where the
shipped result diverged from what the agent first produced — gets one dated
entry here:

```
YYYY-MM-DD-<short-slug>.md
```

The distilled **rule** from a retro lives in the relevant reference doc (this
domain's README, another domain's, or the charter) so the next session reads
it before acting. These entries keep the **context and the WHY** behind that
rule — how it came to be, what the agent did first, and what got parked as
not-yet-durable pending a second occurrence.

Read a rule and want to know why it exists? The retro that produced it is
here.
