# Retros

The reasoning trail for workflow learnings, written by the
[`create-retro`](../skills/create-retro/SKILL.md) skill (`/retro`).

Each session that earned a retro — one with real back-and-forth, or where the
shipped result diverged from what the agent first produced — gets one dated
entry here:

```
YYYY-MM-DD-<short-slug>.md
```

The distilled **rule** from a retro lives in the doc the next session reads —
the [conventions](../conventions/README.md), a skill, or the
[charter](../../AGENTS.md). These entries keep the **context and the WHY**
behind that rule: how it came to be, what the agent did first, and what got
parked as not-yet-durable pending a second occurrence.

Read a rule and want to know why it exists? The retro that produced it is here.
